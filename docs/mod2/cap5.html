Sintaxis y comentarios

Al escribir código, tenemos que seguir correctamente la sintaxis del lenguaje de programación que estemos usando para poder ejecutar la aplicación exitosamente. La sintaxis de un lenguaje de programación se define como el conjunto de reglas que deben seguirse al escribir el código fuente de los programas para considerarse como correctos para ese lenguaje de programación, cada lenguaje tiene su propia sintaxis, cada uno con unas palabras clave o expresiones más u otras menos, pero todos son muy parecidos entre sí. Hay que aprender la sintaxis del lenguaje en cuestión para poder escribir correctamente el programa y pueda ejecutarse, si falta una coma, un punto y coma o hay algún error ortográfico, el programa dará error. Java, como cualquier otro lenguaje de programación, son muy estrictos en cuanto a su sintaxis, ya que son maquinas, no piensan, solo siguen instrucciones, y si las instrucciones están mal escritas, ellas no lo sabrán, por lo que el compilador toma nota y ni siquiera permitirá compilar nuestro código a código máquina.

Los elementos básicos constructivos de un programa son:

  • Palabras reservadas (propias de cada lenguaje).
  • Identificadores (nombres de variables, nombres de funciones, nombre del programa, etc.)
  • Caracteres especiales (alfabeto, símbolos de operadores, delimitadores, comentarios, etc.)
  • Expresiones.
  • Instrucciones.

Aprenderemos de estos elementos a través del curso, así que ponte atento.

Comentarios

En el código no puedes escribir cualquier texto o mensaje que no tenga sentido para la sintaxis, ya que generaría un error de sintaxis y no se podrá compilar el código correctamente (Visual Studio Code puede detectar este tipo de error incluso ante de intentar compilar el código marcándolo con un subrayado rojo, como lo muestra la imagen)

Si quisiéramos poner un mensaje en el archivo para ayudarnos a saber qué hace cierta parte del código o saber para qué sirve aquella sentencia, podemos usar los llamados comentarios, que es la forma que Java nos da de poner mensajes de ayuda en el código. Estos se ponen con dos barras ( // ) al principio del mensaje para que Java ignore todo lo que sigue para que pongas tu texto a gusto hasta el siguiente párrafo, otra forma de ponerlo es con los símbolos de barra-asterisco y asterisco-barra ( /* */ ) para abrir un bloque de comentarios, en el cual, lo que este adentro de este bloque será tratado como comentario y será ignorado por Java. En este curso haremos uso de los comentarios para guiarte en el código y tengas un mejor aprendizaje.
